# 简案自动配置工具

这个子项目和当前已完成的项目分开维护，目录独立，结构沿用现有“插件 Popup + 本地助手 + Excel 配置表”的模式。

## 当前已完成

1. 独立插件目录：`chrome_extension/`
2. 独立本地助手：`local_jianan_assistant.py`
3. 独立配置表：`简案配置表.xlsx`
4. 独立监听脚本：`listen_jianan_requests.py`
5. 独立启动脚本：`run_jianan_assistant.bat`

## 配置表字段

- `源课类ID`
- `源课类名称`
- `源课件代码`
- `目标课类ID`
- `目标课类名称`
- `目标课件代码`
- `简案文件名`
- `下载状态`
- `上传状态`
- `备注`

其中 `源课类ID`、`目标课类ID` 现在可以留空。

- 如果填写了，工具会优先在对应课类里查找。
- 如果留空，工具会按课件代码自动搜索。

## 当前执行逻辑

当前版本已经完成下面这些能力：

1. 创建并维护配置表
2. 在插件中检查本地助手状态
3. 读取配置表并生成待执行任务计划
4. 捕获页面请求中的 `Session-Id`
5. 根据源课件代码读取源课件当前已有的 `jane_case`
6. 根据目标课件代码找到目标课件的简案记录
7. 通过 `plan_edit` 把源课件的 `jane_case` 直接写到目标课件
8. 把执行结果回写到 `简案配置表.xlsx`

也就是说，当前自动化走的是你这次手工验证过的路径：

- 先读取源课件已经存在的简案标识
- 再把这个简案标识复制到目标课件

这条路径不重新上传 Excel 文件，风险更低，也更贴近这次真实操作结果。

另外，当前还支持一条特殊规则：

- 如果来源课件代码没有后缀，例如 `s1_v7_01`
- 则不直接复制来源 `jane_case`
- 而是读取来源课件的 `think_game`（知识链解析）
- 先下载该文件，再重新上传
- 最后把新上传得到的附件 ID 写入目标 `*_TW` 课件的 `jane_case`（课程培训）

## 使用方式

1. 运行 `run_jianan_assistant.bat`
2. 加载 `chrome_extension/` 插件
3. 打开简案页面并刷新一次，确保插件捕获到 `Session-Id`
4. 填写 `简案配置表.xlsx`
5. 先点“检查任务”
6. 再点“开始执行”

## 说明

如果后续出现某些课件不能只复制 `jane_case`、必须重新上传 Excel 文件的情况，再继续接入 OSS 上传那条链路即可。当前监听结果已经把那段上传流程抓到了，后续可以继续补。
